    회사 직군 — 회사를 엔지니어처럼 이해하기¶

    회사 안의 다양한 직군의 업무와 시스템을 소프트웨어 엔지니어링의 언어로 분석합니다.

    핵심 철학¶

    이 시리즈의 목표는 "개발자가 다른 직군의 업무를 대신한다"는 것이 아닙니다. 오히려 "개발자가 다른 직군의 시스템을 이해할 수 있도록 만든다"는 것입니다. 개발자는 자신의 전문성으로 다른 팀과 더 효과적으로 협업할 수 있습니다.

    직군 (직능)¶

    PM (Product Management)¶

    제품의 방향성과 우선순위를 결정하는 체계입니다. - 로드맵: release plan — 제품 개발의 계획과 타이밍 - 스프린트: iteration cycle — 짧은 주기의 실행과 피드백 - 요구사항: spec — 구현해야 할 기능의 명세 - 사용자 피드백: bug report — 실제 사용 경험에서 나오는 신호

    PM 도메인 보기 →

    법무 (Legal)¶

    회사의 법적 위험을 관리하고 규정을 준수하는 시스템입니다. - 계약: interface contract — 양 조직 간의 약속과 책임 - 컴플라이언스: rule set — 따라야 할 규칙과 정책 - 분쟁: conflict resolution — 문제 해결과 완화 프로세스 - 계약 검토: code review — 계약 체결 전 검증

    법무 도메인 보기 →

    재무 (Finance)¶

    회사의 자금 흐름과 재정 건강을 관리하는 시스템입니다. - 예산: resource allocation — 회사 자원의 배분 - 비용: runtime cost — 각 활동의 실행 비용 - 회계마감: batch closing cycle — 주기적 재정 정산 - 성과 분석: profiling — 어느 부문이 효율적인지 분석

    재무 도메인 보기 →

    HR (Human Resources)¶

    회사의 인력 자산을 개발하고 관리하는 시스템입니다. - 채용: candidate pipeline — 인재 발굴과 선별 - 평가: review protocol — 성과 측정과 피드백 - 조직설계: scaling architecture — 회사의 구조와 확장 - 경력개발: skill tree — 인력 성장 경로

    HR 도메인 보기 →

    세일즈 (Sales)¶

    고객 확보와 매출 창출을 위한 시스템입니다. - 리드: input stream — 잠재 고객의 흐름 - 파이프라인: state machine — 고객의 상태 변화 관리 - 전환: conversion event — 리드에서 고객으로의 변환 - 관계 관리: connection protocol — 고객과의 지속적 관계

    세일즈 도메인 보기 →

    마케팅 (Marketing)¶

    회사의 인지도, 신뢰도, 수요를 만드는 시스템입니다. - 브랜드: stable abstraction layer — 회사의 일관된 이미지 - 캠페인: traffic generation — 목표 고객에게 도달하기 - 퍼널: conversion pipeline — 관심에서 구매까지의 흐름 - 분석: analytics — 효과 측정과 최적화

    마케팅 도메인 보기 →

    운영 (Operations)¶

    회사의 일상적 업무와 프로세스를 관리하는 시스템입니다. - SOP: runbook — 반복되는 작업의 표준화 절차 - 물류: logistics pipeline — 제품이나 서비스의 흐름 - 재고: inventory buffer — 시간 변동에 대비하는 버퍼 - 공급망: supply network — 외부 자원을 내부로 가져오기

    운영 도메인 보기 →

    디자인 (Design)¶

    사용자 경험과 인터페이스를 설계하는 시스템입니다. - 디자인시스템: component library — 재사용 가능한 UI 요소 - 사용자리서치: user testing — 실제 사용자의 니즈 파악 - 와이어프레임: blueprint — 제품의 구조 설계 - 프로토타입: mockup — 아이디어를 빠르게 검증

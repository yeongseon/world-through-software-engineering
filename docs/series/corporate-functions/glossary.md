# 회사 직군 용어 사전

이 문서는 회사 직군 시리즈에서 사용하는 도메인 특화 용어들을 정리합니다.

## PM (Product Management)

| 실제 용어 | 메타포 | 설명 |
|----------|--------|------|
| 로드맵 | Release Plan | 제품 개발의 중기 계획과 릴리스 일정 |
| 스프린트 | Iteration Cycle | 2-4주 단위의 짧은 개발 주기 |
| 요구사항 | Spec | 구현해야 할 기능의 명세 |
| MVP | Minimum Viable Product | 가장 기본적인 기능으로 시작 |
| 사용자 피드백 | Bug Report | 실제 사용 경험에서 나오는 신호 |
| OKR | Quarterly Goal | 분기별 목표와 핵심 결과 |
| 우선순위 | Priority Queue | 여러 작업 중 처리 순서 결정 |

## 법무 (Legal)

| 실제 용어 | 메타포 | 설명 |
|----------|--------|------|
| 계약 | Interface Contract | 양 조직 간의 약속과 책임 |
| 컴플라이언스 | Rule Set | 따라야 할 법적 규칙과 정책 |
| 분쟁 | Conflict Resolution | 문제의 해결과 소송 프로세스 |
| 계약 검토 | Code Review | 계약 체결 전 법적 검증 |
| 조항 | Function Parameter | 계약 조건의 각 구성 요소 |
| 위약금 | Exception Penalty | 계약 위반 시의 비용 |
| 배상 | Bug Fix Cost | 문제 해결에 드는 비용 |

## 재무 (Finance)

| 실제 용어 | 메타포 | 설명 |
|----------|--------|------|
| 예산 | Resource Allocation | 회사 자원의 배분과 계획 |
| 비용 | Runtime Cost | 각 활동 실행에 드는 비용 |
| 회계마감 | Batch Closing Cycle | 월/분기/연도 마다의 정산 |
| 성과 분석 | Profiling | 어느 부문이 효율적인지 분석 |
| ROI | Return on Investment | 투자 수익률 |
| 캐시플로우 | Money Flow | 회사의 현금 흐름 |
| 예측 | Forecast | 미래 재정 상황의 추정 |

## HR (Human Resources)

| 실제 용어 | 메타포 | 설명 |
|----------|--------|------|
| 채용 | Candidate Pipeline | 인재 발굴, 지원, 선별 프로세스 |
| 평가 | Review Protocol | 성과 측정과 주기적 피드백 |
| 조직설계 | Scaling Architecture | 회사 구조의 설계와 확장 |
| 경력개발 | Skill Tree | 개인의 성장 경로와 역량 개발 |
| 보상 | Incentive Mechanism | 급여, 보너스, 복리후생 |
| 이직 | Function Migration | 다른 조직으로의 이동 |
| 이탈 | System Failure | 핵심 인력의 퇴사 |

## 세일즈 (Sales)

| 실제 용어 | 메타포 | 설명 |
|----------|--------|------|
| 리드 | Input Stream | 잠재 고객의 흐름 |
| 파이프라인 | State Machine | 고객의 상태 변화 (prospect→customer) |
| 전환 | Conversion Event | 리드에서 고객으로의 변환 |
| 관계 관리 | Connection Protocol | 고객과의 지속적인 관계 |
| 영업 기회 | Task Queue | 처리할 수 있는 기회들 |
| 성약 | Commit | 고객의 구매 의지 |
| 매출 | Output Throughput | 최종 성과로서의 수익 |

## 마케팅 (Marketing)

| 실제 용어 | 메타포 | 설명 |
|----------|--------|------|
| 브랜드 | Stable Abstraction Layer | 회사의 일관된 이미지와 정체성 |
| 캠페인 | Traffic Generation | 목표 고객에게 도달하는 활동 |
| 퍼널 | Conversion Pipeline | 관심에서 구매까지의 흐름 |
| 분석 | Analytics | 효과 측정과 최적화 |
| 세분화 | Segmentation | 고객의 다양한 그룹 분류 |
| 포지셔닝 | Market Position | 시장에서의 차별화된 위치 |
| CTR | Click Through Rate | 광고 클릭율로 본 효율성 |

## 운영 (Operations)

| 실제 용어 | 메타포 | 설명 |
|----------|--------|------|
| SOP | Runbook | 반복되는 작업의 표준화 절차 |
| 물류 | Logistics Pipeline | 제품/서비스의 배송과 전달 흐름 |
| 재고 | Inventory Buffer | 시간 변동에 대비하는 버퍼 |
| 공급망 | Supply Network | 외부 자원을 내부로 가져오기 |
| 효율성 | Throughput | 단위 시간당 처리량 |
| 병목 | Bottleneck | 프로세스 성능을 제한하는 요소 |
| 자동화 | Automation | 수작업을 시스템으로 대체 |

## 디자인 (Design)

| 실제 용어 | 메타포 | 설명 |
|----------|--------|------|
| 디자인시스템 | Component Library | 재사용 가능한 UI 요소 모음 |
| 사용자리서치 | User Testing | 실제 사용자의 니즈와 행동 파악 |
| 와이어프레임 | Blueprint | 제품의 구조와 레이아웃 설계 |
| 프로토타입 | Mockup | 아이디어를 빠르게 검증할 수 있는 형태 |
| UX | User Experience | 사용자가 느끼는 전체 경험 |
| 사용성 | Usability | 시스템이 얼마나 쉽게 사용되는지 |
| 접근성 | Accessibility | 모든 사용자가 사용할 수 있는 정도 |

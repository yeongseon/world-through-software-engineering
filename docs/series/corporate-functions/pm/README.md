# PM — Product Management

제품의 방향성, 우선순위, 그리고 개발 주기를 결정하는 체계를 소프트웨어 엔지니어링의 관점에서 분석합니다.

## 도메인 개요

PM은 비즈니스 요구사항과 기술적 실현 가능성을 연결하는 역할입니다. 소프트웨어 프로젝트의 계획, 실행, 피드백 루프를 관리합니다.

## 주요 은유

- **로드맵**: Release Plan — 제품 개발의 중기 계획과 우선순위
- **스프린트**: Iteration Cycle — 짧은 주기의 실행과 피드백
- **요구사항**: Spec — 구현해야 할 기능의 명세
- **MVP**: Minimum Viable Product — 가장 기본적인 기능으로 시작
- **사용자 피드백**: Bug Report — 실제 사용에서 나오는 신호

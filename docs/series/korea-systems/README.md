# 대한민국 시스템 — 대한민국을 엔지니어처럼 이해하기

대한민국의 정치, 법률, 경제, 사회 제도를 소프트웨어 엔지니어링의 언어로 분석합니다. 이 시리즈는 개발자가 자신이 살고 있는 사회의 시스템을 구조, 인터페이스, 프로세스의 관점에서 이해할 수 있도록 돕습니다.

## 도메인

### 정치 (Politics)
대한민국의 정치 체계를 시스템 아키텍처로 이해합니다.
- **헌법**: root architecture — 모든 다른 시스템의 기반
- **국회**: legislative pipeline — 법안이 처리되는 흐름
- **선거**: leadership rotation — 리더십 교체 메커니즘
- **행정부**: execution runtime — 정책 실행 환경

[정치 도메인 보기 →](/docs/series/korea-systems/politics/README.md)

### 법률 (Law)
대한민국의 법체계를 소프트웨어 아키텍처로 분석합니다.
- **법체계**: layered architecture — 헌법, 법률, 시행령, 규칙의 계층
- **위임입법**: dependency injection — 상위 법이 하위 법에 권한 위임
- **법개정**: changelog — 법의 변경 이력 관리
- **위헌심사**: validation — 법의 유효성 검증 프로세스

[법률 도메인 보기 →](/docs/series/korea-systems/law/README.md)

### 경제 (Economy)
대한민국의 경제 시스템을 소프트웨어 엔지니어링으로 분석합니다.
- **예산**: resource allocation — 국가 자원의 배분
- **세금**: incentive mechanism — 경제 행동에 영향을 주는 메커니즘
- **금리**: control parameter — 경제를 조절하는 주요 변수
- **전세**: unique protocol — 한국만의 고유한 부동산 거래 프로토콜

[경제 도메인 보기 →](/docs/series/korea-systems/economy/README.md)

### 사회 (Society)
대한민국의 사회 제도를 시스템 관점으로 분석합니다.
- **연금**: sustainability system — 장기적인 재정 지속성을 위한 시스템
- **교육**: sorting pipeline — 인재 선발 및 분류 파이프라인
- **출산율**: throughput indicator — 사회 시스템의 주요 성능 지표
- **사교육**: shadow processing — 공식 시스템 외부의 병렬 처리 시스템

# 시리즈 횡단 주제 — Cross-Series Topics

여러 시리즈를 가로지르는 교차 분석을 모으는 공간입니다.

## 이 섹션의 목적

하나의 시리즈 내에서는 깊이 있는 분석을 합니다. 하지만 때로는 **두 개 이상의 시리즈를 연결하는 주제**가 있습니다:

- 정부 예산 vs 기업 예산: 예산 편성과 자원 배분의 상사성과 차이
- 공공 파이프라인 vs 기업 파이프라인: 의사 결정 프로세스의 구조 비교
- 법 시스템 vs 조직 정책: 규칙 체계의 설계 원칙

이런 주제들은 `topics/` 섹션에서 다룹니다.

## 언제 topics를 사용하나?

**Series를 사용할 때:**
- 특정 도메인에 깊이 있게 집중하는 분석
- 예: "국회 입법 프로세스 상세 분석" → korea-systems/politics/
- 예: "PM 로드맵 작성 기법" → corporate-functions/pm/

**Topics를 사용할 때:**
- 두 개 이상의 시리즈를 비교/연결하는 분석
- 예: "정부와 기업의 의사결정 구조 비교"
- 예: "법적 규칙과 조직 정책의 공통점"
- 예: "공공 시스템과 기업 시스템의 인터페이스"

## 현재 상태

아직 topics 항목이 없습니다. 횡단 분석을 제안하고 싶으신가요? [새 글 제안](/.github/ISSUE_TEMPLATE/new-article-proposal.yml)을 해주세요!

## 가능한 주제들 (예상)

- **정부 예산 vs 기업 예산**
  - 자원 배분의 원칙
  - 우선순위 결정 메커니즘
  - 재정 건전성 평가 기준

- **공공 파이프라인 vs 기업 파이프라인**
  - 의사 결정 흐름
  - 이해관계자 관리
  - 실행과 평가

- **법적 계약 vs 조직 계약**
  - 인터페이스 정의
  - 분쟁 해결 메커니즘
  - 검증 프로세스

- **교육 시스템 vs 직원 교육**
  - 인재 개발 파이프라인
  - 평가 메커니즘
  - 경력 경로 설계

더 많은 주제를 제안해주세요!
